        Hello Anthony,

        Thanks! 

        Ever since I learned it in CPT223, it has very quickly become one of my most favoured languages. Before then, I always avoided it because it kind of has a bad reputation, kind of like BASIC; but nowhere near that bad! ...I guess I shouldn't pass judgements based on other peoples opinions and should take the time to find out what I'm missing out on for myself... (On the other hand, if it was labelled as an esoteric language...)

        Since most data I tend to deal with is textual (metaprogramming, web development, system configuration files, other plain text flat files, command-line arguments, general I/O or IPO, etc...), and Perl is designed with text processing in mind; I consider it an imperative language (pun intended). I value Perl just as much as I value Prolog.

        There is still a lot of room for improvement though. For instance, there are some things that I can't do that I like doing in C/C++ (I can't think of a reason why I shouldn't be able to do certain things in terms of logical complexities and algorithmic efficiencies), and I miss Python's "with" statement; especially for piece of mind when it comes to file handling, although I wouldn't trade it for the diamond operator. And I absolutely love how Perl automatically includes features such as regular expressions, command substitution, system calls, pseudo-random number generation, etc... That's something I always found inconvenient about Python; getting caught out for not explicitly importing a fairly common module for something that I tend to find myself using all too often.

        These days I mostly just use the Python interpreter as an advanced calculator; unless I want fractional precision without common rounding errors, in which case I use bc.
